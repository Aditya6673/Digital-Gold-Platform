# Contributing Guide

Thank you for your interest in contributing! This guide outlines a simple, low-friction process.

## Getting Started
- Ensure you are on the latest `main` branch.
- Create a feature branch from `main` for your change.

## Commit Style
- Write clear, descriptive commit messages.
- Prefer small, focused commits.

## Code Quality
- Keep changes minimal and scoped.
- Do not commit secrets. Use `.env` and `.env.example` instead.

## Pull Requests
- Describe the motivation and changes.
- Link related issues if available.
- Ensure lint/build checks (if configured) pass.

## Reporting Issues
- Provide steps to reproduce, expected vs. actual behavior, and environment details.

Thanks for helping improve the project!
